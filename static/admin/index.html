Getting Started

Base URL

https://your-domain.com/api/proxy

Authentication

Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Example Request

curl -X POST https://your-domain.com/api/proxy/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Upstream Provider

All requests are proxied to ai.hackclub.com

CORS Information

What is CORS?

Cross-Origin Resource Sharing (CORS) is a security feature that controls which websites can access your API from a browser.

Common Configurations

Allow All (Development) Origins: *

Allows requests from any domain. Use only for testing.

Specific Domains (Production) Origins: https://app.example.com,https://www.example.com

Restricts access to specific trusted domains.

With Credentials Origins: https://app.example.com
Allow Credentials: Enabled

Allows cookies and auth headers. Origins cannot be *.

Email Whitelist

Manage allowed emails and domains for user registration

Whitelist Configuration

One email per line. Users with these exact emails can register.
One domain per line (include @). Any email ending with these domains can register.

Whitelist Info

How it works

Users can only register if their email matches a whitelisted email OR if their email ends with a whitelisted domain.

Leave both fields empty to allow anyone to register (not recommended for production).

Examples

Allow specific users alice@gmail.com
bob@outlook.com
Allow entire company @company.com
@subsidiary.com
